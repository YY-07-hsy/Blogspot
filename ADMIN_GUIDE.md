## 🚀 快速开始

### 1. 启动博客

```bash
cd Blogspot
```

在浏览器中打开：`http://localhost:5173/admin`

## ✨ 功能介绍

### 📋 文章管理

#### 查看文章列表
- 登录后台后，您可以看到所有已创建的文章
- 每篇文章卡片显示：
  - 文章分类（蓝色标签）
  - 发布日期
  - 文章标题
  - 文章摘要
  - 操作按钮（编辑/删除）

#### 创建新文章
1. 点击顶部的 **"➕ 新建文章"** 按钮
2. 填写文章信息：
   - **标题**（必填）- 文章的主标题
   - **分类** - 文章所属类别（JavaScript、CSS、TypeScript 等）
   - **发布日期** - 自动填充当前日期，可手动修改
   - **摘要** - 显示在首页的简短描述
   - **内容**（必填）- 支持 Markdown 格式的文章正文
3. 使用 **编辑/预览** 标签切换查看效果
4. 点击 **"💾 保存文章"** 保存

#### 编辑文章
1. 在文章列表中找到要编辑的文章
2. 点击 **"✏️ 编辑"** 按钮
3. 修改文章内容
4. 点击 **"💾 保存文章"** 保存

#### 删除文章
1. 在文章列表中找到要删除的文章
2. 点击 **"🗑️ 删除"** 按钮
3. 在确认弹窗中点击 **"确认删除"**
4. ⚠️ 此操作不可撤销，请谨慎确认

### 📥📤 导入/导出

#### 导出文章
- 点击 **"📥 导出文章"** 按钮
- 文章数据将下载为 `blog-articles.json` 文件
到浏览器或 `blog-articles.json` 文件中

## 🔧 部署到生产博客

### 方式一：手动同步（推荐）

1. 在管理后台编辑完所有文章
2. 点击 **"📥 导出文章"** 下载 `blog-articles.json`
3. 将文件重命名为 `articles-data.json`
4. 放在博客项目根目录下
5. 运行同步命令：
   ```bash
   npm run sync:articles
   ```
6. 运行构建命令：
   ```bash
   npm run docs:build
   ```
7. 部署 `docs/.vitepress/dist` 目录

### 方式二：开发环境预览

在开发环境（`npm run docs:dev`）中：
- 文章数据保存在浏览器的 localStorage 中
- 创作完成后，记得导出备份

---

## 💾 数据备份

### 备份建议

1. **定期导出** - 每次完成重要编辑后，点击导出按钮备份
2. **保存多处** - 将导出的 JSON 文件保存在：
   - 本地电脑
   - 云存储（如 GitHub、Google Drive）
   - 版本控制系统
3. **命名规范** - 建议使用日期命名备份文件，如：
   - `blog-articles-2024-03-17.json`

### 恢复数据

1. 在管理后台点击 **"📤 导入文章"**
2. 选择之前导出的 JSON 文件
3. 文章将被导入到列表中

## ⚙️ 技术细节

### 数据存储
- **位置**：浏览器的 localStorage
- **键名**：`blog-articles`
- **格式**：JSON 数组，每篇文章为一个对象

### 文章数据结构
```json

  "id": "article-slug",
  "title": "文章标题",

  "date": "2024年3月17日",
  "excerpt": "文章摘要",
  "content": "# 文章内容\n支持Markdown..."
```

## ⚠️ 注意事项

### 1. 数据持久性
- 数据存储在浏览器 localStorage 中
- 清除浏览器缓存会丢失所有文章
- **务必定期导出备份！**

### 2. 文章 ID 生成
- 新建文章时，ID 会根据标题自动生成
- 标题中的中文和英文会被保留
- 特殊字符会被替换为连字符 `-`
- 如果标题为空，会生成时间戳 ID

### 3. Markdown 限制
- 预览器仅支持基础 Markdown 语法
- 复杂语法（表格、任务列表等）可能无法正确渲染
- 建议在专业 Markdown 编辑器中编写复杂内容

## 🔗 常见问题

### Q: 刷新页面后文章还在吗？
A: 是的，数据保存在 localStorage 中，刷新不会丢失。

### Q: 换了电脑还能看到文章吗？
A: 不能，localStorage 是浏览器本地存储。建议使用导入/导出功能在不同设备间同步。

### Q: 如何将文章发布到生产博客？
A: 使用导出功能生成 JSON 文件，然后运行 `npm run sync:articles` 同步到文件系统。

### Q: 支持多人协作吗？
A: 当前版本不支持，因为数据存储在本地浏览器。如需多人协作，建议使用云端 CMS。

### Q: 可以上传图片吗？
A: 当前版本不支持图片上传。建议使用外部图床或 Markdown 图片语法引用。
